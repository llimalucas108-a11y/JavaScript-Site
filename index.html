História

Criado em
10 dias.

Em 1995, a Netscape contratou Brendan Eich para criar uma linguagem de scripting para o browser Navigator. O resultado mudou a web para sempre.

◆
MAIO 1995
Criação em 10 dias
Brendan Eich cria a linguagem em apenas 10 dias para a Netscape. O nome original era "Mocha", depois "LiveScript".
◆
DEZEMBRO 1995
Rebatizada: JavaScript
Para aproveitar a popularidade do Java (da Sun Microsystems), a linguagem é renomeada para JavaScript — apesar de não ter relação com Java.
◆
1997
ECMAScript — Padronização
A ECMA International padroniza a linguagem com o nome ECMAScript (ES1), garantindo compatibilidade entre diferentes browsers.
◆
2009
Node.js — JS no servidor
Ryan Dahl lança o Node.js, permitindo que JavaScript rode fora do browser, no lado do servidor, revolucionando o backend.
◆
2015
ES6 — A grande modernização
O ECMAScript 2015 (ES6) traz arrow functions, classes, Promises, template literals, destructuring e muito mais — a maior atualização da história da linguagem.
◆
HOJE
A linguagem mais popular do mundo
Por 12 anos consecutivos, JavaScript lidera a pesquisa do Stack Overflow como a linguagem mais usada, presente em 98% dos websites.

Sintaxe

Como JavaScript

Explore os blocos fundamentais da linguagem — da declaração de variáveis até funções assíncronas modernas.

Declaração de Variáveis

JavaScript tem três formas de declarar variáveis, cada uma com comportamentos diferentes de escopo e reatribuição.

  • var — Escopo de função, içado (hoisting). Uso legado, evite.
  • let — Escopo de bloco, pode ser reatribuído. Preferido para variáveis mutáveis.
  • const — Escopo de bloco, não pode ser reatribuído. Padrão para valores fixos.
  • Tipos primitivos: string, number, boolean, null, undefined, symbol, bigint.
  • Tipagem dinâmica — O tipo é determinado em tempo de execução.
variaveis.js 
// ✅ Moderno: const e let
const nome = "JavaScript";         // string
const versao = 2024;               // number
const incrivel = true;             // boolean
const valor = null;               // null
let contador;                      // undefined

// Template Literals (ES6+)
const msg = `Olá, ${nome}! v${versao}`;
console.log(msg);
// → "Olá, JavaScript! v2024"

// Destructuring
const { nome: n, versao: v } = { nome, versao };

// typeof verifica o tipo
console.log(typeof nome);    // "string"
console.log(typeof versao);  // "number"

Funções

Funções são cidadãs de primeira classe em JavaScript — podem ser armazenadas em variáveis, passadas como argumentos e retornadas de outras funções.

  • function declaration — Içada, disponível antes da definição.
  • function expression — Atribuída a uma variável.
  • arrow function — Sintaxe concisa ES6+, sem próprio this.
  • Parâmetros padrão — Valores default quando argumento é omitido.
  • Rest params e spread — ...args para múltiplos argumentos.
funcoes.js 
// Declaração de função
function somar(a, b) {
  return a + b;
}

// Arrow function (ES6+)
const multiplicar = (a, b) => a * b;

// Parâmetro padrão
const saudar = (nome = "Mundo") => {
  return `Olá, ${nome}!`;
};

// Higher-order function
const aplicar = (fn, x, y) => fn(x, y);
aplicar(somar, 3, 4); // → 7

// Rest parameters
const total = (...nums) =>
  nums.reduce((acc, n) => acc + n, 0);

total(1, 2, 3, 4); // → 10

Arrays & Objetos

Arrays e objetos são as estruturas de dados centrais do JavaScript, com métodos poderosos para manipulação funcional de dados.

  • map() — Transforma cada elemento, retorna novo array.
  • filter() — Filtra elementos por condição.
  • reduce() — Acumula valores em um único resultado.
  • Spread operator — ... para copiar e mesclar.
  • Objetos — Pares chave-valor com métodos e herança por protótipo.
arrays-objetos.js 
const nums = [1, 2, 3, 4, 5];

// map — transforma
const dobrados = nums.map(n => n * 2);
// → [2, 4, 6, 8, 10]

// filter — filtra
const pares = nums.filter(n => n % 2 === 0);
// → [2, 4]

// reduce — acumula
const soma = nums.reduce((acc, n) => acc + n, 0);
// → 15

// Objeto com método
const pessoa = {
  nome: "Ana",
  idade: 28,
  apresentar() {
    return `${this.nome}, ${this.idade} anos`;
  }
};

// Spread — clonar e estender
const atualizado = { ...pessoa, cidade: "Lisboa" };

Controle de Fluxo

Estruturas condicionais e loops permitem controlar a execução do código com base em condições e iterações.

  • if / else if / else — Condicional básica.
  • switch — Múltiplas condições sobre um valor.
  • for / while — Loops tradicionais.
  • for...of — Itera sobre iteráveis (arrays, strings).
  • Ternário — Condicional compacta em uma linha.
controle.js 
const nota = 85;

// if / else
if (nota >= 90) {
  console.log("A");
} else if (nota >= 70) {
  console.log("B"); // → "B"
} else {
  console.log("C");
}

// Ternário
const resultado = nota >= 60 ? "Aprovado" : "Reprovado";

// for...of
const frutas = ["🍎", "🍌", "🍇"];
for (const fruta of frutas) {
  console.log(fruta);
}

// switch
switch ("JS") {
  case "JS":
    console.log("JavaScript!");
    break;
  default:
    console.log("Outra linguagem");
}

Async / Await

JavaScript é assíncrono por natureza. Promises e async/await permitem lidar com operações demoradas (I/O, rede) sem bloquear a execução.

  • Promise — Representa um valor futuro: pending, fulfilled, rejected.
  • async — Declara uma função assíncrona que retorna uma Promise.
  • await — Pausa a execução até a Promise resolver.
  • try/catch — Captura erros em código assíncrono.
  • Fetch API — Interface moderna para requisições HTTP.
async.js 
// Promise básica
const esperar = (ms) =>
  new Promise(resolve => setTimeout(resolve, ms));

// async / await com Fetch API
async function buscarUsuario(id) {
  try {
    const resposta = await fetch(
      `https://api.exemplo.com/users/${id}`
    );

    if (!resposta.ok) {
      throw new Error("Usuário não encontrado");
    }

    const dados = await resposta.json();
    return dados;

  } catch (erro) {
    console.error("Erro:", erro.message);
  }
}

// Paralelismo com Promise.all
const [user1, user2] = await Promise.all([
  buscarUsuario(1),
  buscarUsuario(2)
]);

Manipulação do DOM

O DOM (Document Object Model) é a representação em árvore do HTML. JavaScript pode selecionar, criar, modificar e remover elementos dinamicamente.

  • querySelector — Seleciona elementos com seletores CSS.
  • addEventListener — Escuta eventos do usuário.
  • innerHTML / textContent — Modifica o conteúdo.
  • classList — Adiciona, remove e alterna classes CSS.
  • createElement — Cria novos elementos HTML.
dom.js 
// Selecionar elementos
const btn = document.querySelector("#meu-btn");
const lista = document.querySelector(".lista");

// Escutar evento de clique
btn.addEventListener("click", () => {
  // Criar novo elemento
  const item = document.createElement("li");
  item.textContent = "Novo item!";
  lista.appendChild(item);

  // Alterar estilo dinamicamente
  btn.classList.toggle("ativo");
});

// Modificar atributos
const img = document.querySelector("img");
img.setAttribute("alt", "Descrição");
img.style.opacity = "0.5";

// Formulário
const input = document.querySelector("input");
input.addEventListener("input", (e) => {
  console.log(e.target.value);
});

Motores JS

Quem executa
o JavaScript?

Os motores JavaScript são softwares que interpretam e compilam código JS usando JIT (Just-In-Time compilation) para máxima performance.

⚡
V8
Google · Chrome & Node.js
O motor mais rápido e amplamente usado. Compila JS diretamente para código de máquina via JIT. Desenvolvido em C++ e open source.
🦊
SpiderMonkey
Mozilla · Firefox
O primeiro motor JavaScript criado pelo próprio Brendan Eich. Hoje suporta WebAssembly e usa um compilador avançado multi-camada.
🔵
JavaScriptCore
Apple · Safari
Motor da Apple, também conhecido como Nitro. Altamente otimizado para iOS e macOS, com foco em consumo de energia e velocidade.
